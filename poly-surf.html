Overview

In this budget-friendly clone of Subway Surfers, I designed and implemented core gameplay systems to ensure fluid mechanics, replayability, and performance optimization. The game features lane-based movement, dynamic level generation, character customization, and a reactive enemy system â€” all crafted to mirror the feel of an endless runner while being technically efficient and expandable.

Technical Details

  • Object Pooling: Obstacles, coins, and floor tiles are managed through a centralized object pooling system to reduce runtime instantiations and garbage collection overhead.
  • Procedural Level Generation: The game dynamically assembles level chunks at runtime using a modular chunk system. Each chunk is pre-designed and seamlessly connected based on spawn points and markers.
  • Player Movement: Supports 3-lane side switching with smooth transitions, as well as jumping and ducking using a combination of swipe detection and keyboard input. Movement speed increases gradually over time for difficulty scaling.
  • Enemy AI: An enemy character chases the player and adjusts its speed based on the player's velocity and crash events. Upon collisions, the player enters a "recovery" state rather than an immediate game over, allowing for more forgiving gameplay.
  • Missions System: A mission manager tracks tasks like collecting coins or traveling distance, promoting long-term engagement through progression goals.
  • Customization: Players can unlock and select different characters and hoverboards. The customization system loads saved preferences and applies them at runtime.
  • Save System: All progress, including coins, unlocked items, selected skins, and mission states, is saved locally using a JSON-based system powered by Unityâ€™s JsonUtility.Data is serialized and stored securely between sessions.
  • Skate System: The skate system allows the player to activate a hoverboard-like item, similar to the skateboard feature in Subway Surfers. Once activated, the player becomes temporarily invincible to obstacles. Upon collision, the skate is destroyed instead of the player, preventing game-over. Additionally, the skate emits a destruction effect that clears nearby obstacles, giving the player a brief moment to recover and reposition. This system enhances gameplay dynamics and adds a strategic layer to player survival.
  • Input Handling: Both touch (swipe) and keyboard input are supported, abstracted through a custom input manager for cross-platform consistency.
  • UI Integration: Real-time updates of score, distance, coins, and missions are integrated into the UI using Unity's UI system (TextMeshPro and Canvas components).

Technologies Used
