Overview

Fruit Party is a 2D slot machine prototype developed as a technical showcase of slot game mechanics. As the sole developer, I managed every aspect of the project â€” from core system design to implementation and testing. The goal was to create a fully functional and easily adjustable slot game with customizable paylines, bet handling, and visual feedback tools to demonstrate win conditions and balancing logic.




Technical Details

  • Payline Generator System: Designed an editor-friendly 3x5 grid tool that allows developers to create, customize, and save unique paylines. These configurations are stored and dynamically used during spin evaluation.
  • Bet Manager: Manages player bet amounts, validating input and ensuring clean transitions between spins, with built-in error handling for invalid bets.
  • Spin Manager: Controls the full spinning process, from triggering animation to evaluating results, ensuring a fair and consistent user experience.
  • Matching Algorithm: Implements logic to detect symbol matches along active paylines. Connected to the indicator system for visual feedback.
  • Indicator Tool: Highlights winning paylines after a spin, giving users clear, immediate visual confirmation of matching symbols.
  • Quick Spin Feature: Adds an optional gameplay toggle for faster spins, mimicking real-world quick play features for faster engagement.
  • Symbol Probability Balancing: Allows developers to tweak the spawn rates of individual symbols to fine-tune gameplay difficulty and reward frequency.
  • Scatter Chance System: A flexible system to adjust scatter symbol frequency, allowing for fine balancing of bonus triggers or special rounds.

Technologies Used

  • Game Engine: Unity
  • Programming Language: C#
  • Development Tools: Visual Studio, GitHub
  • UI Framework: Unity UI & TextMeshPro
